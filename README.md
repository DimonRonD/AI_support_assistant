# AI Support Assistant

AI-ассистент поддержки с двумя каналами работы:

- `FastAPI` для web/CRM/внешних интеграций;
- `Telegram` для общения пользователей и управления операторами поддержки.

Система поддерживает RAG-базу знаний, режим ручного перехвата диалога оператором, оценку качества, журналирование переписки и выгрузку логов.

## Основные возможности

- **RAG и база знаний:** загрузка файлов `TXT/PDF/MD` в `rag_source`, индексация в ChromaDB, использование контекста при ответе.
- **Память диалога:** учитываются последние 5 сообщений (`SQLite`).
- **Кэш запросов:** повторяющиеся запросы могут обслуживаться из кэша с учетом счетчика обращений.
- **Гибрид AI + оператор:** оператор может перехватить сессию, отвечать вручную и закрывать диалог.
- **Оценка и комментарий:** после завершения диалога пользователь может поставить оценку `1..5`, комментарий доступен только после оценки.
- **Ограничения контента:** длина сообщения пользователя до 250 символов; работает тематический фильтр по темам/синонимам.
- **Полное логирование:** хранение всех сообщений (user/ai/support), статуса диалога, оценки и комментария.
- **Экспорт логов:** CSV-выгрузка за дату, опционально по `session_id` и/или `email`.

## Архитектура

- `app/api.py` — HTTP API.
- `app/telegram_bot.py` — Telegram-бот.
- `app/assistant.py` — бизнес-логика ассистента.
- `app/storage.py` — работа с SQLite.
- `app/rag.py` — индекс и поиск по RAG.
- `run_api.py` — запуск API.
- `run_bot.py` — запуск Telegram-бота.

## Настройка окружения

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Подготовить `.env`:

```bash
copy .env.example .env
```

3. Заполнить переменные:

- `OPENAI_API_KEY` — API-ключ OpenAI.
- `OPENAI_MODEL` — модель ответов (например `gpt-5-nano-2025-08-07`).
- `OPENAI_EMBEDDING_MODEL` — модель эмбеддингов.
- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота.
- `TELEGRAM_SUPPORT_CHAT_ID` — `chat_id` чата операторов поддержки.
- `SQLITE_PATH` — путь к файлу SQLite.
- `CHROMA_PATH` — путь к директории ChromaDB.
- `RAG_SOURCE_DIR` — папка источников RAG.
- `MAX_TELEGRAM_MESSAGE_LENGTH` — лимит длины Telegram-сообщений (обычно `4000`).

## Запуск проекта

### Локально

- API:

```bash
python run_api.py
```

- Telegram-бот:

```bash
python run_bot.py
```

### Через Docker Compose

Доступные профили:

- `api-only` — только API;
- `full` — API + Telegram-бот.

Запуск только API:

```bash
docker compose --profile api-only up --build -d
```

Запуск полного стека:

```bash
docker compose --profile full up --build -d
```

Проверка состояния:

```bash
docker compose ps
```

Для сервиса `api` ожидается статус `healthy`.

Логи:

```bash
docker compose logs -f api
docker compose logs -f bot
```

Остановка:

```bash
docker compose down
```

## API: быстрый сценарий

1. Авторизуйте пользователя через `POST /auth` (`session_id`, `name`, `email`).
2. Если `has_previous_history = true`, покажите историю и спросите, продолжать ли контекст.
3. Если историю нужно сбросить — отправьте `POST /text` с `query = "очистить историю"`.
4. Для общения отправляйте сообщения в `POST /text`.
5. По завершении диалога сохраните `POST /rate` и опционально `POST /comment`.
6. Для просмотра состояния используйте `GET /dialog/{session_id}`.
7. Для web-ответов оператора используйте polling `GET /support/inbox/{session_id}?after_id=N`.

Минимальные примеры:

```bash
curl -X POST "http://localhost:8000/auth" ^
  -H "Content-Type: application/json" ^
  -d "{\"session_id\":\"crm-user-00042\",\"name\":\"Ivan Petrov\",\"email\":\"ivan.petrov@example.com\"}"
```

```bash
curl -X POST "http://localhost:8000/text" ^
  -H "Content-Type: application/json" ^
  -d "{\"session_id\":\"crm-user-00042\",\"query\":\"Здравствуйте\"}"
```

```bash
curl -X GET "http://localhost:8000/support/inbox/crm-user-00042?after_id=0"
```

Подробное описание API: `API_GUIDE.md` и `API_GUIDE_RU.md`.

## Telegram: зоны доступа и команды

Ниже команды разделены по месту использования.

### Команды в обычном чате с ботом (пользовательский чат)

Эти команды выполняются в личном чате пользователя с ботом:

- `/help` — показать список команд.
- `/text` — включить режим общения с ассистентом.
- `/rate <1-5>` — поставить оценку по завершенному диалогу.
- `/comment <текст>` — оставить комментарий (только после `/rate`).

Дополнительно пользователь может писать обычный текст без команды:

- если диалог в контроле `AI`, ответит ассистент;
- если диалог перехвачен поддержкой, пользователь увидит уведомление об ожидании оператора;
- если диалог закрыт, пользователь получит подсказку поставить оценку.

### Команды только для чата поддержки (операторский чат)

Эти команды работают только в чате с `chat_id = TELEGRAM_SUPPORT_CHAT_ID`.

#### Управление диалогами

- `/dialogs` — список диалогов с текущим статусом, контроллером и оценкой.
- `/take <session_id>` — передать управление оператору.
- `/release <session_id>` — вернуть управление AI.
- `/reply <session_id> <текст>` — ответить вручную от имени поддержки.
- `/close <session_id>` — завершить диалог и инициировать сбор оценки.

#### Логи и отчеты

- `/logs` — показать даты, за которые есть логи.
- `/view_log <YYYY-MM-DD> [session_id] [--email <email>]` — выгрузить CSV за дату:
  - без параметров: все сессии за дату;
  - c `session_id`: только конкретная сессия;
  - c `--email`: только указанный email;
  - фильтры можно комбинировать.

#### Управление тематикой (фильтр релевантности)

- `/themes` — список тем.
- `/themes add <тема>` — добавить тему.
- `/themes remove <тема>` — удалить тему.
- `/themes clear` — очистить темы.
- `/synonimic` — показать словарь синонимов.
- `/synonimic add <название> | <синоним1,синоним2>` — добавить синонимы.
- `/synonimic remove <название>` — удалить группу.
- `/synonimic clear` — очистить словарь.

#### Управление RAG и техобслуживание

- `/rag_add` — загрузка файла в `rag_source`.
- `/rag_source` — список файлов-источников.
- `/rag_detail <filename>` — просмотр чанков файла.
- `/rag_load` — индексировать только новые файлы.
- `/rag_reload` — полная пересборка RAG.
- `/rag_clear <filename>` — удалить файл и его чанки.
- `/clear` — очистка памяти сессии (для текущего chat/session).
- `/clear_cache [N YYYY-MM-DD]` — очистка кэша (по умолчанию или по фильтрам).
- `/cache_view` — просмотр кэша.

## Важные правила поведения системы

- **Авторизация API обязательна:** перед `POST /text`, `POST /rate`, `POST /comment`, `GET /dialog/{session_id}`, `GET /support/inbox/{session_id}` нужна `POST /auth`.
- **Комментарий только после оценки:** без `/rate` команда `/comment` вернет ошибку.
- **Лимит пользовательского сообщения:** до 250 символов.
- **Закрытый диалог:** после `/close` новые вопросы не обрабатываются до новой сессии/переоткрытия логики.
- **Web-пользователи и поддержка:** для web-клиента ответы оператора забираются через `/support/inbox`, а не через Telegram.
