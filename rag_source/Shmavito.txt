Проект Shmavito

1) Техническая составляющая проекта
Shmavito - веб-платформа аренды вещей на Django (аналог классифайда/маркетплейса в стиле Avito/eBay).
По текущему состоянию репозитория это не только каркас, а рабочий MVP с основными пользовательскими сценариями.

Тип приложения: монолитный Django-проект.
Основная структура: shmavito (доменная логика), templates (UI), static/media (ресурсы), docker-compose.yml и Dockerfile (запуск).
Точка входа: manage.py.
Маршруты приложения: shmavito/urls.py.
Ключевая логика: shmavito/views.py.
Формы и валидация: shmavito/forms.py.
Модели предметной области: shmavito/models.py.

2) Используемые технологии
Основной стек:
- Python 3.13
- Django 5.2.7
- PostgreSQL
- psycopg2 / psycopg2-binary

Дополнительные пакеты:
- django-admin-interface
- django-colorfield

Инфраструктура:
- Docker
- Docker Compose

Важное замечание:
requirements.txt требует очистки и фиксации минимального набора зависимостей под runtime/CI.

3) Процесс работы проекта (runtime-поведение)
Реализованные сценарии:
- регистрация и авторизация пользователя;
- добавление/редактирование/удаление товара;
- добавление/редактирование/удаление предложения;
- поиск и фильтрация предложений;
- создание заказа на период с расчетом цены;
- отмена заказа с восстановлением доступных интервалов;
- модерация товаров и предложений;
- профиль пользователя, комментарии и рейтинг.

Базовый flow:
1. Пользователь находит подходящее предложение через listing и фильтры.
2. Оформляет заказ, выбирая дату начала и число дней.
3. Система создает заказ и переразбивает интервалы доступности объявления.
4. После сделки пользователь может оставить оценку и комментарий.

4) Процесс развёртывания
В репозитории есть docker-сценарий:
- docker-compose up shmavito_db
- docker-compose up shmavito_web

Контейнер web выполняет:
- migrate
- loaddata data.json
- collectstatic --no-input
- runserver 0.0.0.0:8001

5) Текущие ограничения и риски
Несмотря на функциональность, в репозитории есть блокирующие несоответствия:
- не найден пакет core/settings.py, хотя manage.py ожидает core.settings;
- в docker-команде обязательна загрузка data.json, но файл отсутствует;
- часть шаблонов, используемых во views, отсутствует (add_good.html, add_ad.html, delete_good.html);
- во views есть дублирующийся cancel_order и битая функция comment;
- автотесты фактически отсутствуют (tests.py пустой).

6) Инструкция по работе (текущее понимание)
Для запуска потребуется:
1. Подготовить корректные Django settings (или восстановить пакет core).
2. Проверить источник initial data (data.json) либо убрать жесткую зависимость loaddata.
3. Выполнить миграции.
4. Запустить docker-compose или runserver.
5. Протестировать ключевые сценарии: auth -> listing -> make_order -> cancel_order.

7) Экономические выгоды (бизнес-уровень)
После стабилизации и доведения до production проект дает:
- быстрое выведение на рынок собственной площадки аренды;
- снижение затрат на ручную обработку объявлений, модерацию и учет сделок;
- рост оборота за счет повторных заказов и доверия через рейтинг/отзывы;
- масштабируемую основу для расширения каталога и регионов;
- более высокую операционную эффективность без пропорционального роста штата.
