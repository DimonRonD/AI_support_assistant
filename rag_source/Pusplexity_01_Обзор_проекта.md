# Pusplexity — Обзор проекта

## Назначение

`Pusplexity` — это мультимодальный ассистент для работы с изображениями и корпоративными документами. Проект объединяет Telegram-бота и веб-интерфейс, чтобы пользователи могли:

- редактировать изображения по текстовому описанию;
- генерировать изображения с нуля;
- вести текстовый диалог с моделью;
- получать ответы по документам через RAG (Retrieval-Augmented Generation).

## Бизнес-задача

Проект закрывает типовые задачи команд, которым нужно быстро:

- преобразовывать визуальный контент без ручной обработки в графических редакторах;
- находить информацию в документах без длительного чтения вручную;
- получать единый интерфейс для работы с AI в Telegram и браузере.

## Ключевые возможности

### 1. Работа с изображениями

- Редактирование 1–10 изображений за запрос.
- Команды естественным языком (коллаж, смена стиля, удаление фона и т.д.).
- Отдельные режимы моделей (gpt-image-1, gpt-image-1.5, gpt-image-2-2026-04-21, gpt-image-1-mini).
- Генерация изображения по тексту без исходного файла (`create`, `dalle_gen`).

### 2. Текстовый режим

- Обычный чат с LLM.
- Анализ одного изображения в текстовом режиме.
- Временная память диалога (20 последних сообщений).
- Поддержка контекста из загруженного документа в рамках сессии.

### 3. RAG-база знаний

- Загрузка документов: TXT, PDF, XLSX, XLS, DOCX, MD.
- Индексация в ChromaDB с OpenAI-эмбеддингами.
- Семантический поиск по чанкам документов.
- Ответы модели на основе найденного контекста.
- Список источников, удаление источника и очистка истории RAG-сеанса.

### 4. Web-панель

- Авторизация по email/паролю.
- UI, повторяющий функциональность Telegram-бота.
- Markdown-рендер ответов (списки, заголовки, таблицы, code blocks, ссылки).
- Защита API: CSRF, rate limit, security headers, ограничение upload 50 МБ.

## Каналы доступа

- Telegram (`bot.py telegram`)
- Web (`web_app.py` или `gunicorn web_app:app`)
- CLI для локального теста image-обработки (`bot.py cli`)

## Данные и хранение

- Пользовательская память и выбранные режимы: SQLite (`user_data.db`).
- Логи действий и usage по токенам: SQLite (`action_logs.db`).
- Документы RAG: директория `data/`.
- Векторный индекс RAG: директория `chroma_db/`.
- Telegram persistence: `bot_data.pickle`.

## Технологический стек (кратко)

- Python, Flask, python-telegram-bot
- OpenAI API (chat/image/embeddings)
- ChromaDB
- SQLite
- Docker Compose, Gunicorn, Traefik

## Ограничения и правила работы

- До 10 изображений в image-режимах.
- В text-режиме — не более 1 изображения на запрос.
- Максимальный размер файла в web API — 50 МБ.
- История чата хранится до 20 последних сообщений.
- Ограничение длины промпта: обычно 4000 символов; для DALL-E create-режима — 1000.

## Экономический эффект

- Снижение времени на поиск информации в документах.
- Снижение затрат на типовую визуальную постобработку.
- Рост скорости ответа пользователям и сотрудникам.
- Контролируемое использование AI за счет логов токенов и компонентов.
